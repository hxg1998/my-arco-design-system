# Page Patterns

## List Page

Use a header, filter area, table toolbar, results table, and pagination region.

## Form Page

Use grouped sections, clear required markers, primary and secondary actions, and inline validation.

## Detail Page

Use summary cards at the top, tabbed content when necessary, and action buttons near the title.

## Common Layout Guidance

- Keep page headers sticky only when actions need persistent access.
- Use cards to separate dense information blocks.
- Limit each section to one clear job.
